# AI-Table-Pet-Vision (智能实体桌宠视觉大脑)

本项目是智能 AI 实体桌宠的本地视觉感知与运动控制核心。系统利用摄像头捕获视频流，结合 MediaPipe 提取手部 3D 骨骼关键点，并通过随机森林分类器（Random Forest）实现高精度的多分类手势识别。同时，系统支持串口（UART）与局域网（UDP）双模控制输出，为下位机（如 STM32/ESP32）提供毫秒级的左右追踪偏差与动作触发指令。

---

## 文件清单与功能说明

根据项目目录，各文件的核心功能与说明如下：

*   **`data_collector.py`**
    *   **功能**：数据采集工具。调用摄像头与 MediaPipe 实时提取手部特征，配合键盘按键（0-4）录制不同类别的手势数据。
    *   **使用**：运行后，将手放在镜头前，按住对应的数字键即可持续录制特征，按 `q` 键退出并保存。
*   **`gesture_dataset.csv`**
    *   **功能**：特征数据集文件。由 `data_collector.py` 自动生成，包含 63 列归一化特征数据与 1 列标签（Label）数据。
*   **`model_trainer.py`**
    *   **功能**：模型训练（炼丹）脚本。利用 Pandas 读取 CSV 数据集，划分训练集与测试集，并使用 `scikit-learn` 训练随机森林分类模型。
    *   **使用**：数据采集完成后，直接运行此脚本，控制台会输出模型准确率，并自动生成模型权重文件。
*   **`gesture_model.pkl`**
    *   **功能**：序列化后的模型权重文件。视觉核心程序启动时会加载此文件进行实时推理。
*   **`requirements.txt`**
    *   **功能**：Python 环境依赖清单，精简版，用于在新电脑上一键复现开发环境。
*   **`vision_uart_core.py`**
    *   **功能**：串口直连控制核心。加载模型进行实时推理的同时，以 50Hz 的频率向指定串口发送 4 字节控制帧（包含目标 X 轴偏差与动作指令）。
    *   **使用**：需在代码中修改 `COM_PORT` 为实际连接的端口号，运行即可。
*   **`vision_net_core.py`**
    *   **功能**：局域网 UDP 控制核心。适用于无线分布式架构测试，通过 Socket 网络传输交互指令。
    *   **使用**：需在代码中修改目标硬件的 `ESP32_IP` 与端口号。

## 使用指南

### 1. 环境配置 (Prerequisites)

建议使用 Conda 创建独立开发环境，避免与其他项目冲突：

```bash
# 创建并激活 Python 3.9 环境
conda create -n table-pet python=3.9
conda activate table-pet

# 一键安装核心依赖
pip install -r requirements.txt
```

### 2. 完整工作流 (Workflow)

**第一步：录制手势数据**

Bash

```
python data_collector.py
```

- `[0]` - 负样本 (手乱动、自然下垂)  
- `[1]` - 张开手 (挥手预备)  
- `[2]` - 握紧拳头 (出拳预备)  
- `[3]` - 比心 (触发害羞)
- `[4]` - 伸出食指 (触发好奇

**第二步：训练大脑模型**

Bash

```
python model_trainer.py
```

等待控制台输出准确率（通常 $>95\%$），确认目录下生成了 `gesture_model.pkl`。

**第三步：启动视觉控制端**

确认已通过 USB 数据线连接单片机（或串口模块），并确认端口号无误：

Bash

```
python vision_uart_core.py
```

## 串口通讯协议 (UART)

`vision_uart_core.py` 向下位机发送的数据包采用固定的 **4 字节数据帧** 格式：  

| **字节序号** | **数据定义**         | **数据类型/范围** | **详细说明**                                                 |
| ------------ | -------------------- | ----------------- | ------------------------------------------------------------ |
| **Byte 0**   | **包头 (Header)**    | `0xAA`            | 帧起始标志                                                   |
| **Byte 1**   | **动作指令 (CMD)**   | `0x00` ~ `0x06`   | `0x01`:挥手 | `0x04`:出拳 | `0x05`:指着 | `0x06`:比心 | `0x00`:无动作 |
| **Byte 2**   | **X轴追踪值 (Data)** | `-100` ~ `100`    | 转为 `uint8_t`，带符号位。负数代表手偏左，正数代表手偏右     |
| **Byte 3**   | **包尾 (Tail)**      | `0xFF`            | 帧结束标志                                                   |
